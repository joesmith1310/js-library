This is an example webpage that uses the TeamViewer library. This is a use case where the developer has created one team viewer i.e. only one group of players is being represented, but they have instantiated two formation views for this team. This allows the user to visualise two formations at the same time. The target user of web apps like this could be sports coaches who want to plan before training sessions or a game.

Try clicking on a player node to expand their information. Drag and drop player nodes to swap them around. Drag nodes over to the substitues bench to make a subtitution. You can also add to the group of players being represented using the add player form. Change the sport and see how this affects the number of players on the field and the formations that are available to you. Change the formation to see how the nodes rearrange themselves. When you expand a node you can remove the player from the group.

Note that you can't exchange players across formation views as each player should be shown exactly once in each view.
