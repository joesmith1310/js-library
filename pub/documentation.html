Getting Started

This guide demonstrates how to incorporate TeamViewer.js into a project, add visual TeamViewer components to HTML and then access these elements in javascript

1) Download TeamViewer.js and add it to your project files.


2) Include the script for TeamViewer in the HTML before any scripts that use it:
<script src="TeamViewer.js" type="text/javascript" defer></script>

3) In the script that uses TeamViewer.js initialise a TeamViewer object:
const tv1 = new TeamViewer('viewer1', 'green', 'red');
You must provide a unique id that is associated with this TeamViewer e.g. viewer1


4) In your HTML add a div element where you want a TeamViewer component to appear:
<div class="tv-formation-view viewer1"></div>
<div class="tv-roster-view viewer1"></div>
<div class="tv-player-form viewer1"></div>
The class list of the div should contain the class corresponding to the visual TeamViewer component:

FormationView = tv-formation-view

RosterView = tv-roster-view

PlayerForm = tv-player-form

The class list of the div should also contain the id of the TeamViewer object you want the component to be linked to e.g. viewer1


5) In the script that uses TeamViewer.js create the object referenced in the HTML:
tv1.createFormationViews();
tv1.createRosterView();
tv1.createPlayerForm();

6) Access the objects:
const fv1 = tv1.getFormationView(0);
const rv = tv1.getRosterView();
const pf = tv1.getPlayerForm();

TeamViewer

The TeamViewer is the main component of TeamViewer.js. It is used to represent a group of players. All visual components of TeamViewer.js must be linked to a TeamViewer.

TeamViewer(viewerId, primary = 'gray', accent = 'red')

Constructor for the TeamViewer object.

Parameters

viewerId : String

A unique ID to be assigned to the TeamViewer

primary : String

The primary color of the TeamViewer and the default color for all its visual components

accent : String

The accent color of the TeamViewer and the default accent color for all its visual components

Returns

TeamViewer

The TeamViewer object

Example

const tv1 = new TeamViewer('viewer1', 'green', 'red');

TeamViewer.createFormationViews(width = 500, height = 500, sport = 'Soccer', formation = [1,4,4,2], nodeSize = 70, primaryColor = primaryColor, accentColor = accentColor, landscape = false, editable = true, showSubsBench = true, interactive = true)

Creates all of the FormationView objects associated with this TeamViewer. The FormationViews should have each been indicated in the HTML

Parameters

width: Number

The default width for all FormationViews associated with this TeamViewer (pixels)

height : Number

The default height for all FormationViews associated with this TeamViewer (pixels)

sport : String

Must be in {'Soccer', 'Hockey', 'Football', 'Basketball', 'Baseball', 'Cricket', 'Futsal', 'Lacrosse', 'Rugby', 'Custom'}

formation : [Number]

Initial formation for all FormationViews associated with this TeamViewer e.g. [1,4,4,2]

nodeSize : Number

The initial size of the nodes for all FormationViews associated with this TeamViewer (pixels)

primaryColor : String

The primary color for all FormationViews associated with this TeamViewer. Primary color of a FormationView is the color of the pitch

accentColor : String

The accent color for all FormationViews associated with this TeamViewer. AccentColor of a FormationView is the color of the nodes and buttons

landscape : Boolean

If true all FormationViews associated with this TeamViewer are landscape by default, if not they are portrait

editable : Boolean

If true all FormationViews associated with this TeamViewer each have a tool bar by default which allows the user to edit them. The user can also move players/nodes around and remove players. If not the user is not able to edit the FormationViews by default

showSubsBench : Boolean

If true all FormationViews associated with this TeamViewer each have a section displaying the members of the team who aren't on the pitch. If false this section is hidden

interactive : Boolean

If true the nodes in all FormationViews associated with this TeamViewer are expandable by default

Returns

null

Does not have a return type

Example

tv1.createFormationViews(450, 300, 'Soccer', [1,4,4,2], 50, 'green', '#C70039', true, false, false, false);

TeamViewer.createPlayerForm(width = 500, theme = primaryColor)

Creates a PlayerForm associated with this TeamViewer. The PlayerForm should have been marked in the HTML (Each TeamViewer should only have one or zero PlayerForms)

Parameters

width : Number

The width of the PlayerForm DOM element (pixels)

primaryColor : String

The primary color of the PlayerForm DOM element. In a PlayerForm primary color is the color of the fields and buttons

Returns

null

Does not have a return type

Example

tv1.createPlayerForm(500, '#981717');

TeamViewer.createRosterView(width = 500, maxHeight = 500, theme = primaryColor, ediatble = true)

Creates a RosterView associated with this TeamViewer. The RosterView should have been marked in the HTML (Each TeamViewer should only have one or zero RosterViews)

Parameters

width : Number

The width of the RosterView DOM element (pixels)

maxHeight : Number

The maxHeight of the RosterView DOM element (pixels)

primaryColor : String

The primary color of the RosterView DOM element. In a RosterView primary color is the color of the remove buttons. Note these buttons are only visible if the RosterView is editable.

editable : Boolean

If true the user is able to remove players from the TeamViewer via the RosterView DOM element

Returns

null

Does not have a return type

Example

tv1.createRosterView(500, 400, '#981717', false);

TeamViewer.getFormationView(index)

Returns the nth FormationView object associated with this TeamViewer

Parameters

index : Number

The index of the required TeamViewer. The index of the FormationViews is the order in which they appear in the HTML. The first FormationView is at index 0

Returns

FormationView

The nth FormationView object associated with this TeamViewer or null if there isn't one

Example

const fv1 = tv1.getFormationView(0);

FormationView

The FormationView object must first be obtained using TeamViewer.getFormationView(). A FormationView is a visual component of TeamViewer.js that allows the user to see the players of the associated TeamViewer on a pitch. Each FormationView can have multiple frames (pages) which can each display their own formation, drawings and annotations. Each TeamViewer can have multiple FormationViews associated with it

FormationView.toJSON()

Converts the frame currently being displayed by the FormationView into a JSON string

Parameters

null

Does not accept parameters

Returns

String

The JSON string of the frame

Example

const jsonString = fv1.toJSON();

FormationView.loadJSON(json)

Creates a new frame from a JSON string and adds this frame to the FormationView

Parameters

json : String

The JSON string generated using FormationView.toJSON()

Returns

null

Does not have a return type

Example

fv1.loadJSON(jsonString);

FormationView.cycleFrames()

If a FormationView has multiple frames, the currently displayed frame is changed to the next frame in the frame stack. The frame stack is cyclical

Parameters

null

Does not accept parameters

Returns

null

Does not have a return type

Example

fv1.cycleFrames();

FormationView.setAutoCycle(timer)

If a FormationView has multiple frames, it will cycle through them at a given interval

Parameters

timer : Number

The milliseconds for which each frame is displayed until the frames are cycled. If zero, auto cycling is turned off

Returns

null

Does not have a return type

Example

fv1.setAutoCycle(2000);

FormationView.setOrientation()

Changes the orientation of the FormationView

Parameters

landscape : Boolean

If true the FormationView is set to landscape, if flase the FormationView is set to portrait

Returns

null

Does not have a return type

Example

fv1.setOrientation(true);

FormationView.setFormation(formation)

Changes the formation of the frame currently being displayed by the FormationView

Parameters

formation : [Number]

The new formation e.g. [1,4,4,2]. Note that the sum of the array must equal the number of players in a team of the associated sport .e.g 11 for soccer. If the sport is 'Custom' any number of players is allowed

Returns

null

Does not have a return type

Example

fv1.setFormation([1,2,4,4,2]);

RosterView

The RosterView object must first be obtained using TeamViewer.getRosterView(). A RosterView is a visual component of TeamViewer.js that allows the user to see the players of the associated TeamViewer in list form. Each TeamViewer should only have one RosterView associated with it

RosterView.setPrimaryColor(color)

Set the primary color for the RosterView. In a RosterView the primary color is the color of the remove buttons (Note remove buttons are only visible when the RosterView is editable)

Parameters

color : String

The new primary color for the RosterView

Returns

null

Does not have a return type

Example

rv.setPrimaryColor('green');

PlayerForm

The PlayerForm object must first be obtained using TeamViewer.getPlayerForm(). A PlayerForm is a visual component of TeamViewer.js that allows the user to add players to the associated TeamViewer. Each TeamViewer should only have one PlayerForm associated with it

Set the primary color for the PlayerForm. In a PlayerForm the primary color is the color of the input fields and buttons

Parameters

color : String

The new primary color for the PlayerForm

Returns
